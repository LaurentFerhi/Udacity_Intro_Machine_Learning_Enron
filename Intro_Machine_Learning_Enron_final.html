As part of your answer, give some background on the dataset and how it can be used to answer the project question. Were there any outliers in the data when you got it, and how did you handle those? The Enron case was one of the biggest financial scandals of the past 20 years. Founded in 1985 by Kenneth Lay, later joined by Jeffrey Skilling, Enron had become, in terms of market capitalization, the 7th US company. In december 2001, Enron went to bankruptcy and disintegrated almost overnight as a result of massive fraudulent brokerages over years.

The present analysis aims at defining a machine learning algorithm to identify the POI (Persons of Interest) based on a number of data such as financial records or mail exchanges derived from the Enron corpus. The idea is to find the correct features and machine learning model with adapted tunining to identify a POI.

The data exploration is performed on the data_exploration.py script.

First I checked the shape of the dataset. The initial dataset contains 145 people with 18 identified as POI described by 21 features. Then I converted all the columns in numeric format (email adresses became NaN but we won't use it). Finally I replaced all NaN by zeros.

I checked the number of null values per rows and per columns.

  • The THE TRAVEL AGENCY IN THE PARK row is not relevant for the analysis and should be removed.
  • The LOCKHART EUGENE E row is empty and should be removed.
  • The column 'loan_advances' is nearly empty. This feature won't be used.

There is a significant amount of missing values from each feature in this dataset as we can see on the graph below.

By checking the data graphicaly we can notice that the TOTAL row is an outlier in the dataset and should be removed (this outlier has already been highlighted during Udacity course).

I plotted the correlaction matrix of the dataset (this had to be done after removing the TOTAL row). High Pearson's "r" on features not linked (a priori) can indicate the presence of outliers. High Pearson's "r" were obtained between the following features:

  • total stock value vs exercised stock options: in that case, no outliers apperead, the linear correlation is graphically quite obvious.
  • total payments vs loan advances: this time, there is one outlier, but as this point corresponds to Kenneth Lay (our main POI so to say), we want to keep him in the dataset.
  • to messages vs shared receipt with poi: surprisingly, their seems to be a slight lienar correlation between these two features. No significant outliers are to be removed.

I also made a function to search for outliers by calculating the Z-score of all non null data for each column. The function returned all not POI Z-scores above 3.27 or below -3.27 corresponding to $\alpha$ = 0.001 of the population on a two-tailed test. The following outliers are identified:

Additional features:¶

The number of mail as itself may not be the best representative feature for identifying a POI (the total number of mails may depend also of the hierarchical position of the person). But, we can assume that POI would communicate betwen themselves at a higher rate than other employees. Therefore, it can be interesting to create ratios of messages to (or from) POI on the total number of to (or from) messages. So I added two features :

  • the ratio : mail from_poi_to_this_person / from_messages
  • the ratio : mail from_this_person_to_poi / to_messages

The performance of the reatined model has been assessed with and without these new features:

There is almost no difference between the model with and without these new features. This is confirmed in the paragraph below as only one of these 2 added features is used by the select K best function and with the lowest importance.

Scaling:¶

Scaling is important when using a distance based algorithm. For the different tested algorithms, I used a Standard Scaler (based on Z-scores) when relevant. I also tested the effects of a PCA pre-processing on the different algorithm. As the PCA looks for the maximum variance to determine eigen vectors, and the variance is high for high magnitude features, the scaler was usefull in addition to the PCA. However, as I retained the Gaussian Naive Bayes algorithm (which is not distance based) and without PCA, the scaling had no beneficial effect. In fact the PCA decreased the performance of the algorithm.

In the end the best way I found to select the correct parameters like Scaling and PCA was to run the algorithm with and without, and compare the results.

The algorithm I retained in the end is the Gaussian Naive Bayes which gave the best results. I also tried a decision tree algorithm, a K-means algorithm and an AdaBoost algorithm. The Gaussian Naive Bayes has the best performance and also the best train time. An advantage of Gaussian Naive Bayes is that it is performant on small size training dataset. The decision tree precision score was acceptable, but the recall scores were sometimes lower than 0.3 (depending ot the tuning). I tried to set the GridSearchCV scoring to 'recall'. This way, the classifier parameters are taken to maximize the recall. As a result, the recall score slightly increased but at the expense of precision. By doing so, the global performances of this algorithms with recall based scoring or default scoring were equivalent.

The AdaBoost classifier had correct performances but still lower than Gaussian Naive Bayes on our case.

The K-means clustering shows low performances. By eyes only, it is difficult to identify clusters or local optimums in our dataset. This clustering algorithms may not be well adapted to this dataset.

  1. What is validation, and what’s a classic mistake you can make if you do it wrong? How did you validate your analysis? [relevant rubric items: “discuss validation”, “validation strategy”]

The validation of an algorithm consists in running it on a test set and compare the predictions made to the test labels. To do that, we have to split the dataset in two sets, one for training and one for testing.

One classical mistake that can occur is the overfitting. If the data split produces a train set too different from the test set, the algorithm can be "too well adapted" to the train set and therefore, fails to generalized the predictions from the test set. This overfitting generates high variance.
On the other and, if the algorithm is "too well adapted" to the train set, it may be underfitted and generates high biais.

To balance the biais-variance trade-off, I used a cross validation function which generates folds (train and test splits) randomly. I used the function StratifiedShuffleSplit which is adapted to small size datasets. The advantage of this function is to preserve the percentage of samples for each class. In other words, this function ensures that the number of POI in each test and train set is preserved along all iterations.

I set the number of folds to 100 to ensure a low probability to select several identical folds while keeping an acceptable run time.

  1. Give at least 2 evaluation metrics and your average performance for each of them. Explain an interpretation of your metrics that says something human-understandable about your algorithm’s performance. [relevant rubric item: “usage of evaluation metrics”]

The performance of the algorithm has been evaluated using accuracy, precision and recall evaluation metrics.

  • accuracy: How many predictions have been made correctly ? Accuracy can be expressed as the ratio between correclty predicted items and total items.
  • precision: Of all persons that have been labeled as POI by our algorithm, how many are actually POIs ? Precision can be expressed as the ratio between true positives and the sum of true positive and false positive.
  • recall: Of all persons that are actualy POIs, how many did our algorithm labeled as POIs ? Recall can be expressed as the ratio between true positives and the sum of true positive and false negative.
  • f1: Harmonic mean of recall and precision. It can be expressed by the twice the preduct of precision and recall divided by the sum of precision and recall. It gives a global rating of the algorithm taking into account both precision and recall.

After testing the algorithm with the test_classifier function tester.py script, the results are the following:

Evaluation metric Value
Accuracy 0.86
Precision 0.51
Recall 0.43
f1 0.47

The precision and recall are both above 0.3 as requested by the project specification. The meaning of these metrics can be summarized as:

  • 86% of the predictions have been made correclty.
  • 51% of the person labeled POI by our algorithm are actually POIs
  • 43% of the actual POIs have been labeled POIs by our algorithm.
